Introduction

Git is a version control management system (VCS), a piece of software that developers can install and run locally on their computers. It allows developers to take snapshots (aka "commits") of their work and track changes to their projects. If you take snapshots of incremental changes, you can also track how your thinking evolves. Commit early. Commit often.

Most developers use Github, a website that allows developers to work on the same Git folders (aka "repositories" or "repos"). Once you publish your repositories on Github, you can collaborate with other people on Github repositories (aka "remote repositories" or "remotes"). You can think about it as a Dropbox for Git.

Setting up your repository

To make a copy of an existing Git repo, go through the following steps:

  1. Go to the Github page of the remote repo and "fork" your own copy to your own account
  2. Navigate to your own account page and click the "clone" button on the right-hand side. (Now the page's "url" should be on your clipboard.)
  3. Create a local folder (on your computer) where you want to store your repo
  4. Using command line, navigate to that local folder
  5. Type the command and paste the "url" between the brackets below (brackets not necessary)
    git clone [url]

Publishing Content

To publish a file with content added, go through the following steps:

  1. Check the status of your modified file:
    git status
  2. Begin tracking the file so that it can be staged for commit:
    git add [filename]
  3. Commit the file and add comment to help you remember what you were committing:
    git commit -m "add message here"
  4. You can combine steps 2 and 3 (skip staging) by typing this:
    git commit -a -m "add message here"
  5. Share your progress on Github:
    git push origin master
    where [origin] is your cloned remote and [master] is the branch that you have been working on.

Updating Local Repository

To fetch changes made by your teammates (aka "upstream"), go through the following steps:

  1. Check your remote setup:
    git remote -v
  2. If you don't have your remotes setup:
    git remote add "name" [url]
  3. Fetch the changes:
    git fetch "name"
  4. Merge the changes with your local repo:
    git merge "name"/master
  5. If there is a merge conflict, then open sublime to delete the parts you want to discard and save.
  6. Commit and push:
    git commit -m "add message here"
    git push origin master
