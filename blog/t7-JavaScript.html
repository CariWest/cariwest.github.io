Loops

Loops are universally one of the most useful aspects of any coding language. Looping allows a programmer to automate a task for as many times as necessary. It reduces the lines in a program, which subsequently reduces the opportunity for programming errors. Like most languages, both Ruby and JavaScript use looping regularly â€” but the way those loops are structured differs dramatically.

Looping in JavaScript

There are main loops in JavaScript: the while loop, and the for loop. In short, the while loop repeats the same code block until the specified condition becomes false; the for loop repeats the same code block for a specified number of times. A for loop is really just a specialized while loop, where the loop counts your progress and tests the progress against a condition.

A while loop:

i = 1;
while (i <= 5){
  console.log("Printing " + i);
  i++;
}

A for loop:

for (i = 1; i <= 5; i++) {
  console.log("Printing " + i);
}

These loops do the same thing, but the syntax is a little different. You can see that the for loop is a little shorter and more straightforward. All of the elements needed for the for loop's progression (the starting point, the condition for repeating, and the progression). Conversely, the elements of the while loop are spread out over multiple lines of code.

Of course, a while loop is also useful for loops which aren't controlled by counting. Check out this silly little example, which searches for the first number divisible by three (hint: it comes after two and before four).

divisibleByThree = false;
i = 1;

while (divisibleByThree == false) {
  if (i % 3 != 0) {
      console.log(i + " is not divisible by three.");
      i++;
  } else {
    console.log(i + " is divisible by three!");
    divisibleByThree = true;
  }
}

Looping in Ruby

Like JavaScript, Ruby supports both while and for loops. Unlike JavaScript, Ruby supports many other types of loops which Ruby programmers are welcome to utilize. Every programmer has their preferences, but the loops I use most often in Ruby are #while, #each, and #each_with_index.

A while loop:

while i <= 5
  puts "Printing #{i}"
  i += 1
end

An each loop:

array = [1, 2, 3, 4, 5]

array.each { |i| puts "Printing #{i}"}

An each_with_index loop:

array = [1, 2, 3, 4, 5]

array.each_with_index { |i, index| puts "Index #{index}: #{i}" }

As you can see, #each and #each_with_index are particularly useful when iterating through arrays. These methods are similarly useful for iterating through hashes.

The advantage to looping in Ruby is that there are an exceptional number of pre-defined loops at your disposal as a programmer. Many methods which would take lines and lines of coding already exist! A broad familiarity of the methods which already exist will save you time and patience when programming in Ruby.

It is important to note that while a #for loop does exist in Ruby, it is rarely used. A loop like #times is used more often, which repeats a code block the specified number of times.

JavaScript vs Ruby

It's clear that Ruby has more resources for a programmer in need of loops. However, with some patience, a programmer can create the same types of loops in JavaScript (if it's desired). In the end, it depends on what you want to run and what platform you need to run it on.
